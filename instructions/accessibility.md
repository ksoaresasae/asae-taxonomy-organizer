# Digital Accessibility

All user interface (UI) components must adhere to WCAG 2.2 level AA standards.

https://www.w3.org/TR/WCAG22/

With every major and minor version, check accessibility against the above standards and make necessary corrections prior to deployment.

If a specific directive in development contradicts any of these standards, flag this but do not correct unless directed otherwise. Continue to show any flags at each deployment until fixed or told to permanently log an exception.

For any exceptions, silently keep a record of these, but list and describe them again if requested, offering the opportunity to discontinue the exception and bring the issue back to being a regular flag until fixed.

If requested to "Perform accessibility check", recheck all UI elements against the standards and report any issues, but do not make changes. This is a safety net for things that might get overlooked in the normal pre-deployment fix process.
